About

I am interested in systems programming, operating systems, observability, performance engineering, and the mechanics that make software behave well under real load.

Bio

This site is where I collect longer technical writing, shorter notes, and opinions shaped by time spent around Unix-like systems. I prefer work that reduces mystery: better tooling, better measurement, and cleaner mental models.

Tech Interests

  • Kernel development and operating system internals
  • Linux performance analysis and tracing
  • FreeBSD system design and tooling
  • eBPF, profiling, and observability workflows
  • Networking, schedulers, and memory behavior

Unix Philosophy

I prefer tools that are small, explicit, inspectable, and composable. The site follows the same rule: simple files, readable source, low moving parts, and content that stays usable without client-side machinery.

Open Source

Open source matters because it lets engineers examine the actual mechanism instead of negotiating with a black box. Good public code and documentation shorten the path from confusion to understanding.
